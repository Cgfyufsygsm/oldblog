树的直径

树的直径指的是树上最长的一条链。

性质

下面假定边权非负。

  1. 树的直径可以有若干条，其一定交于一点 ，且 为这些直径的中点。

    平凡，不证。

  2. 到任意节点 最远的节点 一定是直径的某个端点。

    可以考虑调整法。

  3. 树的直径的端点的度数必然是 。

    还是考虑调整法。

  4. 对于两棵树，直径端点分别为 和 ，则在两棵树中间连一条边合并之之后，新树的直径端点一定从 中产生。

    若新树的直径非原树的直径，则其一定过新加的边，根据性质 发现端点一定分别在两棵树的端点。

  5. 如果在一棵树的叶子节点上接一个叶子节点，则最多改变直径的一个端点。

    可以发现这是必然的，懒得证明了。

求法

树的直径一般有两种求法，一种是 dp，一种是贪心。

dp 的话，先钦定根节点，然后令 为 到 子树内叶子的最大距离。然后在 dp 的过程中在子树的根节点合并答案。

dp 做法可以处理边权为负的情况。

而贪心做法由于基于上面的性质 ，所以无法处理边权为负的情况。

具体地，随便找一个点 开始，找到距离 最远的点 ，根据性质 其一定为直径端点，然后再从 开始找到距离 最远的点 ，根据直径的定义， 即为一条直径。

这就是点分治的核心思想之一：“分治”。

然而如果遇到链的话他就寄了。所以每次应该找子树的重心再递归下去。

总结流程：

  • 找到当前树的重心，设为根节点
  • 处理所有经过根的路径
  • 删除根节点
  • 对于所有的子树，重复上述步骤

点分树

简介

一般来说点分治仅可以处理一部分不修改，离线，或单次询问的问题。

注意到点分治劣的原因很大程度上是因为浪费时间在重复的计算重心和统计无用的答案上了。

而若是我们要做的问题与树的结构无关，例如两点间距离，连通块，关键点等，则可以考虑对点分治每次找到的重心建出一棵树高为 的重构树，然后在这棵重构树上解决问题。

这棵重构树就叫点分树。
