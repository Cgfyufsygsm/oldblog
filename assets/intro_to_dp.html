Lead-in：硬币问题

给定若干面值为 111，222，555，101010，202020，505050 和 100100100 的硬币，问凑出 xxx 块钱最少需要多少枚硬币。

这不就是 RMB 的面值吗！根据生活常识，显然是直接拿尽可能大的挨个挨个凑。

这就是贪心（不知道他昨天有没有给你们讲清楚）

  • 所谓贪心，就是只考虑眼前的情况来进行每一步的决策。不难发现我们每次都是选最大的一个面值的硬币去凑，这就是一个典型的贪心思想。

  • 然而，贪心是有局限性的，只是说对于人民币的这套面值来说，这个贪心策略是 OK 的。

  • 我们来看这套面值：{1,5,11}\{1, 5,11\}{1,5,11}。

  • 于是有了一个反例：要凑出 151515，若是选择 15=11+4×115 = 11 + 4\times 115=11+4×1，则需要 555 枚硬币。

  • 然而，选择 15=3×515 = 3\times 515=3×5 只需要 333 枚硬币。

  • 贪心策略的局限性就被体现出来了：鼠目寸光。
  • 凑 151515 的时候，贪心策略选择了面值为 111111 的币。
  • 然后我们就需要去凑 444。

  • f(n)f(n)f(n) 只和 f(n−1)f(n - 1)f(n−1)，f(n−5)f(n - 5)f(n−5) 和 f(n−11)f(n - 11)f(n−11) 有关。

  • 其比搜索快的原因是，搜索需要枚举使用了什么硬币，而我们的这个算法并不关心这个。

  • 要求 f(15)f(15)f(15) 只需要知道 f(14)f(14)f(14)，f(4)f(4)f(4) 和 f(10)f(10)f(10)。其他的都是无关紧要的。

  • 这个做法的优秀之处在于，其舍弃了冗余信息。
  • 我们只需要记录对解决问题有帮助的信息：f(n)f(n)f(n)。

  • 这个问题的正确性在于，想知道 f(n)f(n)f(n)，只需要知道几个更小的 f(c)f(c)f(c)。

  • 求解 f(c)f(c)f(c) 称为求解 f(n)f(n)f(n) 的子问题。

动态规划的定义

  • 这就是DP（动态规划，dynamic programming）.

  • 将一个问题拆成几个子问题，分别求解这些子问题，即可推断出大问题的解。

  • 满足能使用 DP 求解的问题，需要满足两个基本性质：无后效性和最优子结构性。下面进行讲解。

无后效性

  • 回顾一下刚才的问题的一些性质。
  • 我们是不是知道了 f(n−1)f(n - 1)f(n−1)，f(n−5)f(n - 5)f(n−5) 和 f(n−11)f(n - 11)f(n−11) 的值之后，就能计算 f(n)f(n)f(n)，而他们具体怎么计算的就已经无所谓了。
  • 这就是未来与过去无关。

如果给定某一阶段的状态，则在这一阶段以后过程的发展不受这阶段以前各段状态的影响

最优子结构

  • 回顾 f(n)f(n)f(n) 的定义：凑出 nnn 块钱需要的最少硬币数量。
  • 是不是大问题（f(n)f(n)f(n)）的最优解能由小问题的最优解（f(c)f(c)f(c)）推出？
  • 这就是最优子结构性。
  • 一个问题能用 DP 解决当且仅当其满足如上两条性质。

做 DP 题的流程

  • 判断出其为 DP 题。（需要长期大量积类的做题经验和数学直觉）
  • 设计 DP 状态（我是谁）
  • 求出状态转移方程（我从哪来，我要到哪去）
  • 写代码

以上几个流程都是非常重要的，下面会对其进行详细阐述。

DP 的状态设计

  • 上午讲搜索的时候应该提到了，何为状态。
  • 实际上就是当前的一个局面，说的比较正式而已。
  • 上面的那个问题里面，我们将凑出 nnn 块钱作为一个局面，然后设这个局面的最优解（上面就是最少硬币数量）设为 f(n)f(n)f(n)。
  • 以后我们会接触各种各样的状态设计。
  • 总而言之，状态设计就是一个把我们面对的局面转为数学模型的一个过程，这实际上就是一个对建模能力的考查。

状态转移方程

  • 上面那个问题中，我们要求 f(n)f(n)f(n)，是不是需要知道 f(n−1)f(n - 1)f(n−1)，f(n−5)f(n - 5)f(n−5) 和 f(n−11)f(n - 11)f(n−11)。
  • 即 f(n)f(n)f(n) 和这三个局面有关。
  • 我们将使用一个状态的解推出后面状态的解称为状态转移。
  • f(n)=min⁡{f(n−1),f(n−5),f(n−11)}+1f(n)=\min\{f(n - 1), f(n - 5), f(n - 11) \}+ 1f(n)=min{f(n−1),f(n−5),f(n−11)}+1 就是所谓的状态转移方程。

  • 所以令 fi,jf_{i,j}fi,j​ 表示当前走到了 (i,j)(i,j)(i,j) 时的最优答案。
  • 状态设计这一步就完成了。
  • 考虑转移。假设我们要求 fi,jf_{i,j}fi,j​，则我们可以从 fi−1,jf_{i -1, j}fi−1,j​ 或 fi−1,j−1f_{i - 1, j -1}fi−1,j−1​ 得来。

#include <bits/stdc++.h>
using namespace std;

const int maxn = 1005;
int a[maxn][maxn], f[maxn][maxn], n, ans;

int main() {
    cin >> n;
    for (int i = 1; i <= n; ++i) {
        for (int j = 1; j <= i; ++j) {
            cin >> a[i][j];//读入数字三角形
        }
    }
    for (int i = 1; i <= n; ++i) {
        for (int j = 1; j <= i; ++j) {
            f[i][j] = max(f[i - 1][j], f[i - 1][j - 1]) + a[i][j];
        }
    }
    for (int j = 1; j <= n; ++j) ans = max(ans, f[n][j]);
    cout << ans << endl;
    return 0;
}

最长上升子序列（LIS）问题

给定长度为 nnn 的序列 aia_iai​，从 aaa 中抽一个子序列出来，要求其单调递增。问最长的 LIS 长度。

例：1,5,3,4,6,9,7,81,5,3,4,6,9,7,81,5,3,4,6,9,7,8 的 LIS 为 1,3,4,6,7,81,3,4,6,7,81,3,4,6,7,8，长度为 666。

需要一个 O(n2)O(n^2)O(n2) 的算法。

洛谷上对应的题为导弹拦截，但今天这个方法过不了。

状态设计

  • 我们发现，递增的子序列中，重要的是最后一个元素，其他的都无关紧要。
  • 所以设 fif_ifi​ 表示以 aia_iai​ 结尾的 LIS 的长度。
  • 然后我们发现这个状态定义的还是比较不错的，即满足无后效性也满足最优子结构。

状态转移

  • 这题的转移就没有那么的简单了。
  • 我们不妨考虑将 aia_iai​ 接到之前的某个 LIS 上面去，设其为 jjj。
  • 则，若 aj<aia_j < a_iaj​<ai​，则 fif_ifi​ 可以由 fj+1f_j + 1fj​+1 转移过来。

背包 DP

是一类很重要的 DP 问题。大体可以抽象为一个容量有限的背包，每样物品有其容量和价值，要求装这个背包使得得到的价值最大。根据具体问题的不同分为 0-1 背包，完全背包，分组背包，泛化背包，树上背包等等。

今天先掌握 0-1 背包和完全背包。

可以百度搜索崔添翼大神的背包九讲的 pdf 下来学习。

0-1 背包

有 NNN 个物品和一个容量为 VVV 的背包，第 iii 个物品的体积为 CiC_iCi​，价值为 WiW_iWi​。现在求装物品能得到的最大价值。（洛谷 P1048 采药）

可以贪心吗？

  • 啊，我们不妨按照价值最大的挨个挨个选。反例应该很好找吧。
  • 按照体积从小到大呢？也是错的好吧。
  • 如果按照价值与体积的比值从大到小来选呢？
  • 我告诉你，错的。
  • 考虑背包容量为 100100100，有三个物品（用 (Wi,Ci)(W_i, C_i)(Wi​,Ci​) 表示）： (99,51)(99, 51)(99,51)，(50,50)(50, 50)(50,50) 和 (50,50)(50, 50)(50,50)。
  • 如果按照上面的策略来选，肯定先选第一个，然后你发现你放不了了。
  • however，你选第二个和第三个会更优。寄。

问题分析

  • 首先我们注意到，每个物品要么放要么不放，所以叫 0-1 背包。
  • 然后我们考虑定义状态。可以设 fif_ifi​ 为只考虑前 iii 个物品能得到的答案。
  • 然后你就写不出来状态转移方程了。

去除后效性

  • 因为你会发现，背包的容量是一个影响因素，你不能不考虑背包的容量。
  • 即，其具有后效性。
  • 有后效性的东西，我们考虑将其写在状态里面！

转移方程

  • 所以，改进后的正确状态为：fi,jf_{i,j}fi,j​ 表示考虑前 iii 种物品，背包占用容量为 jjj，能得到的最大价值。
  • 则我们对于一个 fi,jf_{i,j}fi,j​ 的计算无非就是考虑第 iii 件物品取还是不取。
  • 那么，方程就来了：fi,j=max⁡{fi−1,j,fi−1,j−Ci+Wi}f_{i,j}= \max\{f_{i- 1, j}, f_{i - 1, j - C_i} + W_i\}fi,j​=max{fi−1,j​,fi−1,j−Ci​​+Wi​}。
  • 这是一个很重要的方程，请各位务必对其理解透彻！

转移方程的理解

  • fi−1,jf_{i - 1, j}fi−1,j​ 即表示我们当前不装这个物品，所以 dp 值直接继承上一个。
  • fi−1,j−Cif_{i - 1, j - C_i}fi−1,j−Ci​​ 表示，前 i−1i - 1i−1 个物品，背包占用了 j−Cij - C_ij−Ci​ 的情况，因为我们既然加了这个容量为 CiC_iCi​ 的物品，背包占用就变成 jjj 了。
  • 有没有什么不懂的地方？请及时提出来。

问题描述

有 NNN 种物品和一个容量为 VVV 的背包，第 iii 种物品的体积为 CiC_iCi​，价值为 WiW_iWi​。每种物品可以取无限个。现在求装物品能得到的最大价值。（洛谷 P1616 疯狂的采药）

问题分析

  • 这次我们要考虑的，无非是第 iii 个物品取几个罢了。
  • 所以状态转移方程如下：fi,j=max⁡k=0,kCi≤V−j{fi−1,j−kCi+kWi}f_{i,j} = \max_{k = 0, kC_i\le V - j}\{f_{i - 1, j - kC_i} + kW_i\}fi,j​=maxk=0,kCi​≤V−j​{fi−1,j−kCi​​+kWi​}
  • 时间复杂度为 O(NV∑VCi)O(NV\sum\frac{V}{C_i})O(NV∑Ci​V​)，较高。
  • 过不了洛谷那道题。

简单有效的优化之二进制拆分

  • 考虑将第 iii 个物品的 2k2^k2k 次方打包在一起，如果我们将该物品用二元组 (Ci,Wi)(C_i, W_i)(Ci​,Wi​) 表示的话，那我们可以将其拆为以下的 log⁡2VCi\log_2\frac V{C_i}log2​Ci​V​ 个物品：

  • 这是一个比较有用的优化，因为你不管取多少个物品，其都可以转为二进制的形式。
  • 然而还是过不了洛谷那道题。

更牛逼的做法

  • 0-1 背包中，滚动数组的写法是 for (int j = V; j >= c; --j) f[j] = max(f[j], f[j - c] + w);。
  • 我们将其改为 for (int j = c; j <= V; ++j) f[j] = max(f[j], f[j - c] + w); 即可。
  • 只有枚举的顺序发生了改变，怎么回事呢？

该做法的理解

  • 首先我们回顾为什么 0-1 背包要倒序枚举。
  • 那么你应该可以知道为什么完全背包需要正序枚举了。
  • 有不懂的地方提出来。
  • 然后下了课去做题去。

  • 我们之前学习了深度优先搜索（DFS）
  • 实际上，几乎所有的 DP 题都可以使用搜索等价地完成。

给定 R×CR\times CR×C 的矩阵，每个数字代表该地的海拔。我们滑雪可以从某点滑到上下左右四个相邻点之一，且海拔要减小。问最长的滑雪路径。1≤R,C≤1001\le R,C\le 1001≤R,C≤100。

思路分析

  • 我们可以设 fx,yf_{x, y}fx,y​ 为起点为 (x,y)(x, y)(x,y) 能滑的最大路径的长度。
  • 然后我们显然可以知道 fx,yf_{x, y}fx,y​ 可以由四个相邻的点转移而来，即 fx,y=max⁡x′,y′,ax′,y′<ax,y{fx′,y′}+1f_{x, y} = \max_{x', y', a_{x',y'} < a_{x,y}}\{f_{x', y'}\} + 1fx,y​=maxx′,y′,ax′,y′​<ax,y​​{fx′,y′​}+1。
  • 但是你会发现，我们似乎没法很方便的进行 DP。。。。。

搜索

  • 那么我们就考虑搜索吧。
  • 从 (x,y)(x, y)(x,y) 开始 DFS，每次找可以走的更低的点往下走，这样便可以很轻松的搜出来答案。
int dfs(int x, int y) {
    int ret = 1;
    for (int k = 0; k < 4; ++k) {
        int tx = x + fx[k], ty = y + fy[k];
        if (!check(tx, ty) || a[tx][ty] >= a[x][y]) continue;
        ret = max(ret, dfs(tx, ty) + 1);
    }
    return ret;
}

优化

  • 当然，上面这个最裸的搜索铁定是跑不过去的。
  • 究其原因，是因为一个 (x,y)(x, y)(x,y) 我们可能访问很多很多次。
  • 我们注意到，对于 (x,y)(x,y)(x,y)，其答案是确定的，如果我们第二次，第三次……搜到 (x,y)(x,y)(x,y)，我们是否可以利用已经计算好了的答案呢？
  • 那必须是可以的。
  • 这就是记忆化搜索的精髓：保存已经计算好了的信息防止重复计算。
int dfs(int x, int y) {
    if (f[x][y]) return f[x][y];//如果已经计算过了，那么就直接返回结果
    int ret = 1;

        int tx = x + fx[k], ty = y + fy[k];
        if (!check(tx, ty) || a[tx][ty] >= a[x][y]) continue;
        ret = max(ret, dfs(tx, ty) + 1);
    }

在一个圆形操场的四周摆放 NNN 堆石子，现要将石子有次序地合并成一堆，规定每次只能选相邻的2堆合并成新的一堆，并将新的一堆的石子数，记为该次合并的得分。

区间 DP 的一般分析

  • 考虑定义 fi,jf_{i,j}fi,j​ 为 [i,j][i,j][i,j] 的答案（不完全适用，若有别的需要加入状态的维度则需要一并加入）。

  • 答案一般为 f1,nf_{1,n}f1,n​。

  • 发现是一个环，所以考虑断环为链。
  • 设 fi,jf_{i,j}fi,j​ 表示合并了 [i,j][i,j][i,j] 的石子后得到的最大得分（最小得分是类似的，不展开讲了）

  • 合并了 [i,j][i,j][i,j] 石子的得分如何计算？
  • 我们在考虑所有 DP 的转移的时候，都是考虑最后一步。
  • 最后一步显然是将 [i,k][i, k][i,k] 和 (k,j](k,j](k,j] 合并。
  • 得到的得分应为 sj−si−1s_j - s_{i - 1}sj​−si−1​。

#include <bits/stdc++.h>
using namespace std;

const int maxn = 105;
int a[maxn << 1], f1[maxn << 1][maxn << 1], f2[maxn << 1][maxn << 1], n, s[maxn << 1];

int main() {
    scanf("%d", &n);
    for (int i = 1; i <= n; i++) {
        scanf("%d", a + i);
        a[n + i] = a[i];
    }
    for (int i = 1; i <= n << 1; i++)
        s[i] = s[i - 1] + a[i];
    for (int l = 1; l < n; l++)
        for (int i = 1, j = i + l; j < n << 1 && i < n << 1; i++, j = i + l) {
            f2[i][j] = 0x3f3f3f3f;
            int d = s[j] - s[i - 1];
            for (int k = i; k < j; k++) {
                f1[i][j] = max(f1[i][j], f1[i][k] + f1[k + 1][j] + d);
                f2[i][j] = min(f2[i][j], f2[i][k] + f2[k + 1][j] + d);
            }
        }
    int ans1 = 0, ans2 = 0x3f3f3f3f;
    for (int i = 1; i < n; i++) {
        ans1 = max(ans1, f1[i][i + n - 1]);
        ans2 = min(ans2, f2[i][i + n - 1]);
    }
    printf("%d\n%d\n", ans2, ans1);
    return 0;
}
